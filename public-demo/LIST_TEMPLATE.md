# Team Planning Data

Use this generic list structure for a public demo.

| Column | Type | Notes |
|--------|------|-------|
| Task | Single line of text | Main activity title |
| Assigned to | Choice | Worker or assignee |
| Day | Choice | Monday to Friday |
| Week | Date and Time | Use the Monday of the selected week |
| Status | Choice | Pending, In Progress, Paused, Completed |
| Type | Choice | Activity Type 1, Activity Type 2, Activity Type 3, Activity Type 4, Activity Type 5, Activity Type 6 |
| Batch reference | Single line of text | Optional numeric reference |
| Notes | Multiple lines of text | Optional instructions |
| Start time | Date and Time | Start timestamp for tracking |
| End time | Date and Time | End timestamp for tracking |
| CompletedBy | Single line of text | Optional |
| CompletedDate | Date and Time | Optional |

## Suggested defaults

- List name: `Team Planning Data`
- Assignees: `Worker 1` to `Worker 8`
- Keep the list as an app datastore rather than a user-facing list
- Let users work from the planning workspace page instead of editing the list directly
