## Recommended version

I want to share a small project that genuinely changed how I think about SharePoint, VS Code, and AI-assisted development.

I was not coming into this as someone with complete end-to-end knowledge of all the tools used to build the app.

What I did know very well was:

- what the team was doing before in SharePoint
- where the friction was in the current process
- what I wanted the experience to become after

Starting from that, I used Codex in VS Code to explore what was actually possible for this use case.

The result was a shift from a basic SharePoint list view into a lightweight planning workspace with a much better day-to-day experience.

What I found valuable is that I did not need to begin as an expert in every technical layer. Once the operational need was clear, Codex helped me understand the applicable scope, move faster in implementation, and bridge the gap between the current process and the solution I had in mind.

This also reminded me that SharePoint can go much further than a raw list UI when the underlying need is clear.

To make this useful for others, I prepared a free public-safe starter resource with:

- anonymized sample data
- before and after visuals
- a story script
- a simple browser-based demo runner
- a Codex starter guide for applying the same approach in VS Code

If you work in SharePoint and have ever thought “this process should be easier than this”, that was exactly the starting point here.
